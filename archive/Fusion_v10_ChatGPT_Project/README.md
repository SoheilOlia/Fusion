# SeniorCare - Mobile Health App Design

## Vision Statement
*"Technology should disappear into the background, leaving only what matters: your health, your peace of mind, and your connection to care."*

## Core Philosophy: Jobs' Approach to Senior Healthcare

### The Tension: Simplicity vs Functionality
- **Simplicity Demands**: Large buttons, clear text, minimal steps, intuitive navigation
- **Healthcare Requires**: Comprehensive tracking, emergency features, medication management, doctor communication

### Our Resolution: Progressive Disclosure
*"Start with what they need most. Hide complexity until they're ready for it."*

## The Three-Tap Rule
Every essential action must be achievable in three taps or less:
1. **Emergency** → One tap (always visible red button)
2. **Medication Check** → Two taps (home → pills)
3. **Call Doctor** → Two taps (home → doctor)
4. **Health Check-in** → Three taps (home → health → today)

## Design Principles

### 1. Invisible Complexity
- Advanced features exist but don't overwhelm
- Smart defaults handle 80% of use cases
- Progressive disclosure reveals features when needed

### 2. Emotional Design
- Warm, reassuring colors (not clinical white)
- Encouraging language ("Great job!" not "Task completed")
- Human-centered photography

### 3. Graceful Degradation
- Works perfectly even if user only uses 20% of features
- No feature dependencies that break the core experience
- Offline-first design for reliability
